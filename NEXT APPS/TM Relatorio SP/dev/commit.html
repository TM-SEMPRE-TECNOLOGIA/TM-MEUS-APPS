TM Relatório SP - Development Timeline

Setup inicial e Workspace

Passo 1

Duplicação do projeto TM Relatório para TM Relatório SP, inicialização do log de desenvolvimento e documentação de tarefas.

Setup

Backend: Extração de Medidas SP

Passo 2

Implementação do módulo Utils SP para realizar Regex nos nomes de arquivos do modelo organizado, extraindo automaticamente LARGURA, ALTURA e DESCONTO.

Backend

Motor de Varredura Hierárquica

Passo 3

Criação do generator_sp.py para detectar e agrupar inteligentemente as fotos originais e suas subpastas de detalhes (Ex: Foto 1 com - Detalhes 1), compondo um modelo JSON descritivo.

Backend

Integrador Word (Tabelas e Textos)

Passo 4

Substituição do inseridor genérico de imagens do modelo Tradicional por um montador automatizado que gera Tabelas do Word (5 colunas) e junta parágrafos técnicos de manutenção (item do contrato e recusa de cofre).

Backend

Integração API + Frontend Next.js

Passo 5

Atualização da FastAPI e do portal UI Next.js para possuir botões de chaveamento (toggle) entre modo 'Tradicional' e 'Organizado SP', roteando para a engine correta.

Frontend

Validação da Estrutura

Passo 6

Sincronização bem-sucedida entre o motor SP e os dados da Vila Tatetuba, garantindo o emparelhamento de Fotos x Detalhes.

Verificação

Robustez e Algoritmo os.walk

Passo 7

Refatoração completa do motor para usar o algoritmo os.walk do app original. Resultado: Zero perda de dados, atingindo o match de 439 imagens idêntico ao modelo tradicional.

Backend

Refinamento Estético Final

Passo 8

Implementação de Tabelas de Alta Fidelidade (Subtotal/Total), priorização de 'Vista Ampla', destaque de medidas em vermelho e inclusão das tabelas do Item 13.12.

Final
